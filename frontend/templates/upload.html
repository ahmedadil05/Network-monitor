Supported formats: .csv, .txt, .log

Supported data types: connection-level network logs in either: (1) compact 10-column schema, or (2) full NSL-KDD 43-column schema.

Dataset Requirements

The system accepts publicly available network log datasets.

Recommended dataset: NSL-KDD (available from UCI Machine Learning Repository)

Expected columns include:

  • duration, protocol_type, service, flag
  • src_bytes, dst_bytes
  • land, wrong_fragment, urgent
  • label (attack type or normal)

Important: Raw PCAP and firewall text dumps are not directly analyzed. Convert them to supported connection-level CSV fields first.

Parser support: `.log` / `.txt` files with key-value patterns (e.g., duration=1 protocol=tcp src_bytes=500 ...) are auto-converted to compact CSV before anomaly detection.

The system preprocesses the file automatically: parsing → filtering → normalization → anomaly detection → storage.
